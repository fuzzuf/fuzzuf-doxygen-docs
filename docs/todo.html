Class fuzzuf::hierarflow::CallRandomChild< I >

This should derive from HierarFlowIrregularRoutine instead of HierarFlowRoutine after HierarFlowIrregularRoutine is implemented.

Fuzzuf should unify its class designs of random number generators and distributions so that all the classes would not uncontrollably define its random number generator and distributions in their own way. After the unification, CallRandomChild should follow the sole design to define its distribution. Currently, we use std::default_random_engine with ordinary modulus.

This routine can be more useful if it can deal with other distributions than the uniform distribution.
