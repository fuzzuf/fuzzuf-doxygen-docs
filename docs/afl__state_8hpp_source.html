45 // NOTE: we should consider applying "Type Erasure" to this structure due to the
46 // following reason: Let's consider the situation where some variant of AFL uses
47 // "struct AFLDerivedState : public AFLStateTemplate<AFLDerivedTestcase>".
48 // Then, AFLState(=AFLStateTemplate<AFLTestcase>) cannot be not the base class
49 // of AFLDerivedState. This brings a bit troublesome result: we need to
50 // "templatize" almost everything related to (AFL)Testcase and (AFL)State. This
51 // is not only annoying, but also sometimes even harmful in the point that many
52 // functions can be unnecessarily duplicated because of template.
