◆ WriteTestInputToFile()

void fuzzuf::executor::Executor::WriteTestInputToFile ( const u8 *  buf,
u32  len 
)

Precondition:

  • input_fd is a file descriptor that points a file of fuzz. Postcondition:
  • Write out the data pointed by buf to the file pointed by input_fd.
  • The written out file only contains data pointed by buf.
  • The size of written out file is smaller or equal to the value specified by len.
  • Seek "file position indicator" to head of the file for reading the file from target process. Check if the current execution path brings anything new to the table. Update virgin bits to reflect the finds. Returns 1 if the only change is the hit-count for a particular tuple; 2 if there are new tuples seen. Updates the map, so subsequent calls will always return 0.

This function is called after every exec() on a fairly large buffer, so it needs to be fast. We do this in 32-bit and 64-bit flavors.
